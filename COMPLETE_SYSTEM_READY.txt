================================================================================
                          HOW TO START (3 STEPS)
================================================================================

STEP 1: Transfer to Raspberry Pi
---------------------------------
Copy this entire folder to your Raspberry Pi using:
- USB drive, or
- SCP: scp -r meeting-transcriber pi@<pi-ip>:~/
- Git clone

STEP 2: Install (ONE Command on Pi)
------------------------------------
cd ~/meeting-transcriber
chmod +x install_rpi.sh
./install_rpi.sh

Wait 15-20 minutes for automatic installation.

STEP 3: Start Backend (ONE Command on Pi)
------------------------------------------
./run.sh

That's it! Backend is now running and listening.

================================================================================
                        CONFIGURE LAPTOP (2 Commands)
================================================================================

On your laptop:

1. Get Pi IP (on Pi terminal):
   hostname -I

2. Configure frontend:
   cd frontend
   ./configure_frontend.sh    (or .bat on Windows)
   
3. Build and start:
   npm install
   npm run build
   npm run serve

4. Open browser:
   http://localhost:3000

================================================================================
                          TROUBLESHOOTING
================================================================================

Problem: Backend won't start
Solution: Check logs with: tail -f /tmp/backend.log

Problem: No microphone
Solution: Run: arecord -l
          Add to audio group: sudo usermod -a -G audio $USER

Problem: Frontend can't connect
Solution: Check Pi IP: hostname -I
          Update: frontend/.env.production
          Rebuild: npm run build

Problem: Empty transcript
Solution: Test mic: arecord -d 5 test.wav && aplay test.wav
          Increase volume: alsamixer (F4 for capture)

Problem: Summary fails
Solution: Switch to textrank in recorder_config.yml

For detailed troubleshooting, see:

================================================================================
                          AUTO-START ON BOOT
================================================================================

To start backend automatically when Pi boots:

sudo nano /etc/systemd/system/meeting-backend.service

Paste:
[Unit]
Description=Meeting Transcriber Backend
After=network.target

[Service]
Type=simple
User=pi
WorkingDirectory=/home/pi/meeting-transcriber
ExecStart=/home/pi/meeting-transcriber/start_backend.sh
Restart=always

[Install]
WantedBy=multi-user.target

Enable:
sudo systemctl enable meeting-backend
sudo systemctl start meeting-backend
